# TS-GRP-139: Recurring-generated expense missing from full Expenses list

## Description
Recurring-generated expenses are visible on the Dashboard's recent feed but are missing from the full Expenses list ("Show all").

## Finding / Root Cause
This is a client-side caching issue. The web frontend configures a global 1-minute `staleTime` for React Query. 
- The **Dashboard** recent feed fetches expenses using a direct, raw `fetch` call inside a `useEffect` (`await listExpenses(0, 50)`), bypassing React Query entirely. Therefore, it always fetches fresh data from the backend and immediately shows newly created recurring expenses.
- The **Expenses page** uses React Query (`useInfiniteQuery({ queryKey: ['expenses'] })`). When navigating from the Dashboard to the Expenses page, React Query serves the stale cached data (which doesn't contain the new expense generated by the backend cron job) for up to 1 minute.

Since recurring expenses are generated asynchronously on the backend, the frontend receives no cache invalidation signal.

## Proposed Fix
Manually invalidate the React Query cache for `expenses` (and `all-group-expenses`) whenever the "Run now" action is invoked, or configure a shorter polling/stale time for the expenses list to ensure background cron-generated expenses appear promptly.
